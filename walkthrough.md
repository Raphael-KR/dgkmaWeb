# 기능 구현 및 검증 가이드

이 문서는 동국대학교 한의과대학 동문회 웹사이트를 위해 구현된 기능들과 이를 검증하는 방법을 설명합니다.

## 구현된 기능

### 1. 헤리티지 섹션 (동문회 소개)
- **URL**: `/heritage`
- **구성 요소**:
  - **연혁**: 동문회의 주요 역사 타임라인.
  - **조직도/임원**: 임원진 소개 및 조직도.
  - **역대회장**: 역대 동문회장 리스트.
  - **회칙**: 동문회 회칙 및 규정.
  - **역사관**: 사진 갤러리 (예시 데이터).
- **검증 방법**: 하단 메뉴나 사이드 메뉴에서 "동문회 헤리티지"로 이동합니다. 탭(연혁, 조직도, 역대회장, 회칙, 역사관)을 클릭하여 내용이 올바르게 표시되는지 확인합니다.

### 2. 통합 게시판
- **URL**: `/boards`
- **기능**:
  - 카테고리 기반 필터링이 가능한 통합 `posts` 테이블 사용.
  - 카테고리: 안내, 자유게시판, 경조사알림, 당근마켓, 임상 정보 공유방.
  - 카테고리 선택을 통한 게시글 작성.
- **검증 방법**: "게시판"으로 이동합니다. 탭을 눌러 카테고리가 전환되는지 확인합니다. "글쓰기" 버튼을 눌러 게시글 작성 다이얼로그가 열리는지 확인합니다.

### 3. 부고 알림 시스템 (핵심 기능)
- **URL**: `/obituary` (목록), `/obituary/new` (작성)
- **기능**:
  - **목록 보기**: 고인 성함, 관계, 별세 일시, 빈소 정보를 카드 형태로 표시.
  - **AI 자동 입력**: "부고 알리기" 페이지에서 문자 내용(예: SMS)을 붙여넣고 "분석"을 누르면 폼이 자동으로 채워짐.
  - **직접 입력**: 상세 부고 정보를 직접 입력하는 폼 제공.
- **검증 방법**:
  1. 사이드 메뉴에서 "부고 알림"으로 이동합니다.
  2. "부고 알리기" 버튼을 클릭합니다.
  3. 샘플 부고 문자를 붙여넣어 AI 파싱을 테스트하거나 직접 내용을 입력합니다.
  4. 등록 후 목록에 정상적으로 나타나는지 확인합니다.

### 4. 동문 명부
- **URL**: `/directory`
- **기능**:
  - 검색 가능한 동문 리스트.
  - `/api/alumni` 엔드포인트에서 실제 데이터 조회.
  - 이름, 기수, 학과 정보 표시.
- **검증 방법**: "동문 명부"로 이동합니다. 검색창에 이름을 입력하여 결과가 필터링되는지 확인합니다.

### 5. 관리자 및 데이터 동기화
- **URL**: `/admin`
- **기능**:
  - **가입 승인 대기**: 신규 회원 가입 요청 승인/거절.
  - **Google Sheets 동기화**: 기존 Google Sheets의 동문 데이터를 새로운 PostgreSQL 데이터베이스로 동기화.
- **검증 방법**: 관리자 계정으로 로그인합니다(해당되는 경우). "동문 데이터" 탭에서 동기화 기능을 실행해 봅니다.

## 기술적 검증

### 빌드 확인
다음 명령어를 실행하여 프로젝트가 정상적으로 빌드되는지 확인합니다:
```
npm run build
```

### 데이터베이스 스키마
데이터베이스 스키마에 다음 테이블들이 추가/업데이트되었습니다:
- `obituaries` (부고)
- `alumni_database` (동문 DB)
- `posts` (태그 포함 게시글)
- `users` (역할 포함 사용자)
